# 00.project.md

## Project Goal

This example shows how a Python machine-learning project uses `docs/cx` for use cases, tasks, and changes.

## Technical Constraints

- Python commands use a `uv` managed interpreter.
- Training, data, and model objects use full object-oriented design by default.
- Unit tests are not default deliverables; when this example task explicitly declares unit tests, they use `unittest` and mirror the `src` structure.
- Default execution edits one production code file after the task document is complete; split another task before editing a second code file.
- The project-root `AGENTS.md` navigates the training-configuration tutorial; read it before entering the configuration domain.
