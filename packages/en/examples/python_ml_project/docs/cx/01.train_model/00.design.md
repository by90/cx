## Design Decisions

| Decision | Reason | Impact |
| --- | --- | --- |
| Express default behavior with a config object | Avoid script parameter branching | Explicitly declared unit tests can construct the config object directly |

## Implementation Boundary

- The current example task covers the training config class in one production code file.
- This example task explicitly declares a unit test, so the unit-test file mirrors the source path.
