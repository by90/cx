# 教程：使用训练配置

## 教程目标

让训练代码直接构造 `TrainingConfig` 读取默认参数，并在需要时通过构造参数明确覆盖当前训练值。

## 适用读者和场景

- 编写训练入口、模型训练器或数据加载器的开发者。
- 需要调整学习率、批量大小等训练参数的维护者。

## 前置条件

- 已使用 `uv` 建立项目 Python 环境。
- 当前任务已经实现 `src/config/training_config.py` 中的 `TrainingConfig`。

## 公共入口和源码位置

- 公共入口：`src.config.training_config.TrainingConfig`
- 源码位置：`src/config/training_config.py`

## 输入与输出

| 项目 | 内容 |
| --- | --- |
| 输入 | 可选的训练配置构造参数 |
| 输出 | 包含当前训练参数的 `TrainingConfig` 实例 |

## 最小可运行示例

```python
from src.config.training_config import TrainingConfig

config = TrainingConfig()
print(config.batch_size)
```

预期结果：输出配置类定义的默认批量大小。

## 操作步骤

1. 从项目根包导入 `TrainingConfig`。

   预期结果：Python 能解析公共配置入口。

2. 直接调用 `TrainingConfig()` 创建默认配置。

   预期结果：实例包含训练所需的全部默认参数。

3. 训练对象通过实例属性读取参数。

   预期结果：训练入口不解析命令行参数，也不维护另一套默认值。

## 常见失败及处理

| 现象 | 原因 | 正确处理 |
| --- | --- | --- |
| 导入失败 | 没有从仓库根包使用绝对导入 | 使用 `from src.config.training_config import TrainingConfig` |
| 训练参数出现两套默认值 | 调用方重复定义配置 | 删除调用方默认值，直接使用配置对象 |
| 脚本要求命令行参数 | 绕开项目配置入口 | 把行为定义为有默认值的配置字段 |

## 关键约束

- 训练脚本不接收命令行参数。
- 默认值由配置对象统一定义，调用方直接创建配置实例。
- 不为旧配置入口保留兼容层。
