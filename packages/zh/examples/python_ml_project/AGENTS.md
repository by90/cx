# AGENTS.md

本文件是全局 `AGENTS.md` 的项目补充，只登记 Python 机器学习示例项目的事实、边界和必读教程。全局规则继续有效。

## 项目目标

1. 展示 Python 机器学习项目如何用 `docs/cx` 管理训练用例、固定任务和临时变更。
2. 展示训练代码如何通过配置对象读取默认行为，避免命令行参数和重复配置体系。

## 语言与工具链

| 项目 | 当前选择 | 常用命令或入口 |
| --- | --- | --- |
| 编程语言 | Python | `uv run python` |
| 环境和包管理 | `uv` | `uv sync` |
| 测试 | Python `unittest`，仅在任务明确声明时使用 | `uv run python -m unittest discover -s tests` |
| 文档 | 简体中文 Markdown | `docs/cx/` |

## 项目结构与边界

- 生产源码位于 `src/`，测试在明确声明时位于镜像结构的 `tests/`。
- 训练、数据和模型状态使用明确对象表达。
- Python 脚本行为来自配置对象，不接收命令行参数。

## 通用包教程导航

| 领域 | 通用包或稳定能力 | 教程链接 | 公共入口 | 必须先读的场景 |
| --- | --- | --- | --- | --- |
| 训练配置 | `config` | [训练配置教程](docs/cx/docs/01.训练配置教程.md) | `src.config.training_config.TrainingConfig` | 新增或修改训练参数、训练入口或模型默认值前 |

## 领域先读门禁

1. 进入训练配置领域前先读取上表教程。
2. 搜索教程登记的源码入口和真实调用点。
3. 配置入口缺失或教程失效时先用 `$cx-doc` 修正文档和导航。
4. 已有配置对象满足需求时直接使用，不建立第二套配置体系。

## 验证与交付

- 运行当前任务明确指定的最窄验证命令。
- 交付前使用 `$cx-review` 检查实现、教程、链接和完成证据。
