# 教程：发现并使用通用能力

## 教程目标

让调用者在新增实现前找到项目已经登记的通用包和稳定公共入口，阅读正确教程并直接复用现有能力。

## 适用读者和场景

- 准备新增类、函数、组件、数据结构或工具的维护者。
- 准备调用、修改或抽取通用包的人工智能代理。
- 需要维护协议、数据过程、特征体系或指标口径的文档作者。

## 前置条件

- 项目根目录存在按实际项目填写的 `AGENTS.md`。
- `docs/cx/docs/00.索引.md` 已登记当前教程。

## 公共入口和源码位置

- 教程与项目导航入口：`$cx-doc`，源码位于 `SKILLS/zh/cx-doc/SKILL.md`。
- 通用能力发现入口：`$cx-common-module`，源码位于 `SKILLS/zh/cx-common-module/SKILL.md`。
- 流程入口：`$cx-workflow`，源码位于 `SKILLS/zh/cx-workflow/SKILL.md`。

## 输入与输出

| 项目 | 内容 |
| --- | --- |
| 输入 | 当前任务所属领域、要解决的问题和候选实现 |
| 输出 | 被采用的现有公共入口，或确有必要的新入口及其调用者教程和项目导航 |

## 最小可运行示例

```text
任务涉及配置读取。
打开项目 AGENTS.md 中“配置”领域的教程链接。
按教程实例化配置公共入口。
搜索该入口的真实调用点并复用。
```

预期结果：任务直接使用项目已有配置入口，不产生第二套配置体系。

## 操作步骤

1. 从项目 `AGENTS.md` 判断当前任务所属领域，并打开该领域列出的全部教程。

   预期结果：取得当前有效的公共入口、输入输出、约束和验证方式。

2. 搜索教程登记的源码位置和真实调用点。

   预期结果：确认教程没有失效，并找到可以复用的调用方式。

3. 比较现有入口与当前目标；能够满足时直接复用。

   预期结果：实现不新增重复类、包装层或相似局部逻辑。

4. 现有能力确实不能满足时，设计最小新入口，并同时创建教程、更新专题索引和项目 `AGENTS.md`。

   预期结果：新能力有明确调用入口、真实使用场景和可执行教程。

## 常见失败及处理

| 现象 | 原因 | 正确处理 |
| --- | --- | --- |
| 项目 `AGENTS.md` 没有当前领域 | 项目导航未更新 | 先使用 `$cx-doc` 登记领域和教程链接 |
| 通用包存在但没有教程 | 能力交付不完整 | 先补齐调用者教程，再继续相关实现 |
| 教程入口与源码不一致 | 公共入口变化后未同步 | 更新原教程和真实调用点，不保留旧入口 |
| 未搜索就准备新建实现 | 忽略现有能力 | 使用 `$cx-common-module` 完成源码和调用点搜索 |

## 关键约束

- 文档缺失不表示代码能力不存在，仍须搜索源码。
- 每个通用包只有一份当前调用者教程，其它文档只链接，不复制正文。
- 新抽象必须隐藏真实复杂度或服务真实调用点，不为未来猜测增加入口。
- 正式教程只说明当前入口，不保存迁移历史。

## 验证方式

- 核对项目 `AGENTS.md`、教程、源码入口和真实调用点一致。
- 搜索重复实现、绕开通用包的调用和失效链接。
- 使用 `$cx-review` 审查教程可执行性和复用结论。
