# 专题：通用能力与专题文档

## 解决的问题

避免人工智能代理忽略项目已经存在的通用包、稳定接口和技术结论，重复实现相似代码。

## 适用范围

- 通用包、可复用组件和稳定公共接口。
- 协议、数据格式、数据读取过程和字段语义。
- 特征体系、指标口径、模型输入和技术路线。

## 当前结论

工作开始时先读取 `docs/cx/docs/00.索引.md` 和相关专题文档，再搜索专题登记的源码入口与真实调用点。已有能力满足需求时直接复用，不得另写相似实现。

每个通用包和稳定技术过程在 `docs/cx/docs/` 下拥有一份独立编号专题文档。专题文档是详细接口和使用方式的唯一说明；其它文档只链接它。

## 功能入口或源码位置

- `SKILLS/zh/cx-doc/SKILL.md`
- `SKILLS/zh/cx-common-module/SKILL.md`
- `SKILLS/zh/cx-workflow/SKILL.md`

## 输入与输出

- 输入：当前任务涉及的业务概念、技术主题和候选实现。
- 输出：采用的现有入口，或确有必要的新通用入口及对应专题文档。

## 关键约束

- 文档缺失不表示代码能力不存在，仍需搜索源码。
- 新抽象必须隐藏真实复杂度或拥有多个真实调用点。
- 专题文档只说明当前入口，不保存迁移历史。

## 最短使用方式

1. 读取专题索引和相关主题。
2. 搜索公共入口和调用点。
3. 直接复用满足需求的能力。
4. 新增稳定能力时创建下一个编号专题文档并更新索引。

## 验证方式

- 核对专题文档与源码入口一致。
- 搜索重复实现和绕开通用包的调用。
