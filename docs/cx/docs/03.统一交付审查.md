# 专题：统一交付审查

## 解决的问题

避免交付物质量审查和完成证据检查重复执行，同时确保任务不会在缺少验证或文档不一致时被宣布完成。

## 适用范围

- 代码、文档、教程、研究、设计和流程交付物。
- 当前任务状态、验证证据和临时变更删除条件。

## 当前结论

所有交付审查由 `$cx-review` 完成。审查分为两个连续阶段：第一阶段按交付物类型检查质量；第二阶段检查审查覆盖、验证证据、当前状态一致性、临时变更删除条件和剩余风险。

任一阶段失败时任务保持未完成，当前变更文件继续存在。两个阶段通过后，才允许删除当前变更文件。

## 功能入口或源码位置

- `SKILLS/zh/cx-review/SKILL.md`
- `packages/zh/.codex/agents/cx-review.toml`

## 输入与输出

- 输入：用户请求、当前状态文档、实现、验证结果和未完成变更。
- 输出：发现、审查范围、已验证证据、缺失证据、审查结论和剩余风险。

## 关键约束

- 每类交付物分别审查。
- 未运行或失败的验证不能写成通过。
- 正式文档不得保存新旧差异或完成变更。
- 删除变更文件后不能丢失当前有效知识。

## 最短使用方式

1. 列出本轮交付物类型。
2. 执行分类质量审查。
3. 执行完成证据门禁。
4. 失败则修复；通过则删除当前变更文件。

## 验证方式

- 全仓搜索独立证据审查技能和调用。
- 核对审查输出包含质量、证据和剩余风险。
