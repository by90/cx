# 专题：双语包同步

## 解决的问题

确保中文源、英文源、安装包镜像、模板、说明和全局规则表达同一套当前流程。

## 适用范围

- `SKILLS/zh/` 与 `SKILLS/en/`。
- `packages/zh/` 与 `packages/en/`。
- 根目录中文和英文说明。
- 全局 `AGENTS.md` 安装模板。

## 当前结论

中文规则是修改顺序的第一侧。先完成并审查中文技能源、中文安装包、中文模板和中文说明，再同步同等结构与约束到英文侧。

公共技能源与安装包中的技能镜像必须逐字节一致。两个语言包的技能名称和目录结构一致，语言只改变面向人的表达。

## 功能入口或源码位置

- `SKILLS/zh/`
- `SKILLS/en/`
- `packages/zh/manifest.json`
- `packages/en/manifest.json`
- `tools/install_cx_zh.ps1`
- `tools/install_cx_en.ps1`

## 输入与输出

- 输入：通过中文审查的当前规则。
- 输出：结构一致、语义等价的双语源和安装包。

## 关键约束

- 不在本机全局技能目录直接维护最终结果。
- 修改完成后提交并推送 `main`。
- 推送后从远端运行对应安装脚本更新全局技能和 `AGENTS.md`。

## 最短使用方式

1. 修改并审查中文源和中文包。
2. 同步英文源和英文包。
3. 比较技能集合、清单和镜像哈希。
4. 推送 `main`。
5. 从远端安装中文包并核对结果。

## 验证方式

- 比较两个语言包的技能目录名称。
- 比较公共源和安装包镜像的文件哈希。
- 运行包验证工具。
- 安装后核对全局技能和 `AGENTS.md`。
