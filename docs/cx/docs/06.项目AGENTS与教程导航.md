# 教程：建立项目 AGENTS 与教程导航

## 教程目标

为项目建立根目录 `AGENTS.md`，让它根据项目目标、编程语言、工具链和通用包补充全局规则，并明确每个领域必须先读的教程。

## 适用读者和场景

- 初始化 `cx` 项目的维护者。
- 项目语言、目标、工具链或通用包发生当前状态变化时的维护者。
- 进入新业务或技术领域前需要定位现有能力的人工智能代理。

## 前置条件

- 已安装 `$cx-doc`。
- 已能读取项目目标、主要语言、包管理器、构建入口、目录结构和现有通用包。

## 公共入口和源码位置

- 模板入口：`$cx-doc` 的 `assets/AGENTS.md`。
- 中文源：`SKILLS/zh/cx-doc/assets/AGENTS.md`。
- 中文安装包镜像：`packages/zh/.agents/skills/cx-doc/assets/AGENTS.md`。

## 输入与输出

| 项目 | 内容 |
| --- | --- |
| 输入 | 项目目标、语言、工具链、目录结构、命令、通用包和教程 |
| 输出 | 项目根目录中按真实情况填写的 `AGENTS.md` |

## 最小可运行示例

```text
复制 $cx-doc/assets/AGENTS.md 到项目根目录。
把 TODO 替换为项目真实目标、Python 与 uv 工具链、源码目录和验证命令。
为 config 通用包登记教程链接、公共入口和“修改模型配置前”先读条件。
```

预期结果：代理处理模型配置前能从项目 `AGENTS.md` 打开 config 教程，并按教程使用现有入口。

## 操作步骤

1. 读取全局规则和项目当前说明，确认项目目标、语言、工具链、目录和命令。

   预期结果：取得只属于当前项目的事实，不复制全局纪律。

2. 将 `$cx-doc` 的 `assets/AGENTS.md` 复制到项目根目录，并替换所有 `TODO`。

   预期结果：项目规则能说明目标、工具链、结构、项目边界和验证方式。

3. 枚举项目内每个通用包、可复用组件和稳定公共接口，为它们确认独立调用者教程。

   预期结果：每项能力都有当前教程；缺失项先用 `$cx-doc` 补齐。

4. 在项目 `AGENTS.md` 中登记领域、通用包、教程相对链接、公共入口和必须先读的场景。

   预期结果：维护者和代理可以从一个导航表进入正确教程。

5. 打开每个链接，并搜索教程登记的源码入口和真实调用点。

   预期结果：所有链接有效，教程与当前实现一致。

## 常见失败及处理

| 现象 | 原因 | 正确处理 |
| --- | --- | --- |
| 项目规则复制大量全局内容 | 没有区分全局纪律与项目事实 | 删除重复规则，只保留项目补充 |
| 导航只写包名，没有教程链接 | 无法执行领域先读门禁 | 增加仓库相对链接、公共入口和触发场景 |
| 一个通用包没有教程 | 通用能力交付不完整 | 先创建调用者教程并更新索引 |
| 模板仍有 `TODO` | 未按真实项目完成初始化 | 读取项目文件并逐项替换，不猜测事实 |

## 关键约束

- 项目 `AGENTS.md` 不能削弱或覆盖全局规则。
- 教程链接使用仓库相对路径。
- 进入相关领域前必须先读导航中的全部匹配教程，再规划、设计或编码。
- 项目目标、语言、工具链或通用包变化时直接更新原项目规则，不保留旧版。

## 验证方式

- 确认项目根目录存在 `AGENTS.md` 且没有未填写的 `TODO`。
- 检查每个通用包都有教程，并由项目规则导航。
- 打开全部教程链接，核对公共入口和真实调用点。
- 使用 `$cx-review` 审查项目规则完整性和教程可执行性。
