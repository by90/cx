# 变更：禁止顺手处理任务外问题

## 状态

## 当前事实

全局规则和 `$cx-workflow` 已限制默认单任务、单生产代码文件推进，但没有绝对禁止人工智能代理发现任务外问题后“顺手”修改，也没有要求把任务外问题先列成清单并取得用户逐项授权。部分极简、清理和最新意图规则可能被代理错误解释为可以扩大当前任务范围。

## 目标状态

所有编码工作只处理用户当前请求、当前用例路径、原任务和未完成变更共同界定的当前任务。任何任务外问题，无论代理认为多明显、多简单、多严重或多适合顺手处理，都绝对禁止修改；代理只能列出文件、位置、现象、风险和建议，等待用户对具体清单项明确授权。代理自己的推测、偏好、审查意见和所谓最佳实践不构成授权。

## 主要变化

1. 在中英文全局 `AGENTS.md` 铁律中增加绝对禁止顺手处理任务外问题的规则。
2. 在中英文 `$cx-workflow` 执行纪律中把任务边界设为修改前门禁。
3. 在中英文 `$cx-review` 代码审查中把任何未经授权的任务外修改判定为审查失败。
4. 明确任务外问题只能进入待授权清单，清单必须包含文件、位置、现象、风险和建议。
5. 明确只有用户对具体清单项的授权才能扩展任务范围，代理推测和审查意见都不能替代授权。
6. 先修改中文规则和中文安装包，再同步等价英文规则和英文安装包。
