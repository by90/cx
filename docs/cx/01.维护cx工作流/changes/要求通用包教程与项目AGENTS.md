## 当前事实

- 通用包专题文档只要求说明入口和最短使用方式，没有完整的调用者教程结构。
- 安装包提供全局 `AGENTS.md` 模板，没有供项目根目录使用的默认 `AGENTS.md` 模板。
- 项目规则没有要求依据编程语言、项目目标和实际通用包生成项目级补充规则。
- 项目级规则没有集中列出通用包教程链接，也没有按业务领域建立先读文档门禁。

## 目标状态

- 每个通用包在 `docs/cx/docs/` 下拥有一份站在调用者角度、可以按步骤执行的教程。
- `$cx-doc` 负责创建、维护、索引和审查通用包教程。
- 安装包在 `$cx-doc` 技能资产中提供默认项目 `AGENTS.md` 模板，项目初始化时依据编程语言、项目目标和实际通用包生成根目录 `AGENTS.md`。
- 项目 `AGENTS.md` 作为全局规则的补充，列出所有通用包教程链接，并要求进入相关领域前先阅读对应教程。

## 主要变化

1. 把通用包专题文档从接口说明扩展为调用者教程，并补齐前置条件、步骤、最小示例、预期结果和失败处理。
2. 为 `$cx-doc` 增加默认项目 `AGENTS.md` 模板资产和生成维护流程。
3. 在工作流、项目初始化、通用能力发现和统一审查中增加项目级规则与教程阅读门禁。
4. 先更新中文源和中文包，再同步英文源、英文包、示例和说明。
