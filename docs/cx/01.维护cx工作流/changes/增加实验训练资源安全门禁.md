# 变更：增加实验训练资源安全门禁

## 状态

## 当前事实

`$cx-pytorch-tdd`、`$cx-pytorch-quick-hpo` 和 `$cx-pytorch-full-hpo` 已要求记录部分运行期资源指标，但没有统一要求在任何实验训练启动前先评估系统内存和显存占用，也没有明确隔离高资源、长耗时任务与 Codex App 内置终端。

## 目标状态

三个技能都把资源安全作为实验训练的第一道前置门禁。任何实验训练先评估预计系统内存和显存占用；预计资源占用较多或耗时较长的任务在外部终端启动，以持续写入的文件型日志作为观察和分析入口，避免训练任务或终端处理拖垮系统，尤其避免 Codex App 崩溃。

## 主要变化

1. 在三个中文技能开头增加统一、醒目的资源安全前置门禁。
2. 要求启动前记录资源估算、可用资源、安全余量和启动决定。
3. 要求高资源或长耗时任务通过外部终端运行，不在 Codex App 内置终端承载训练进程。
4. 要求训练进度、资源指标和错误持续写入文件型日志，代理通过读取日志文件分析，不依赖终端滚动输出、终端控制序列或特殊终端处理。
5. 中文源和中文安装包通过审查后，同步等价英文规则到英文源和英文安装包。
