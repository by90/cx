## 当前事实

- `$cx-pytorch-quick-hpo` 使用约十分之一完整实体样本，训练上限为 `60` 个 epochs，早停忍耐为 `8`，并向全量调参推荐 `3` 个候选。
- `$cx-pytorch-full-hpo` 使用完整数据，训练上限为 `600` 个 epochs，早停忍耐为 `20`，先训练第 `1` 名候选并等待用户确认是否继续第 `2`、第 `3` 名候选。
- 两个技能要求记录随机种子和 `val_loss` 改进次数，但没有把配置值 `3407` 固定为所有调参实验的共同种子，也没有把最佳轮次是否接近训练末尾作为首要持续性证据。
- 轻量调参没有要求对最终每个候选都执行消融实验和回测；全量调参仍允许在首个候选后停止。

## 目标状态

- 轻量调参固定使用完整实体样本的十分之一、最多训练 `120` 个 epochs、早停忍耐为 `9`，并统一读取配置值 `3407` 作为种子。
- 轻量调参承担数据参数、模型结构参数和训练参数的主要选择，确定 `5` 个候选后，对每个候选分别完成消融实验和回测。
- 全量调参固定使用全部样本，沿用 `120` 个 epochs、早停忍耐 `9` 和配置种子 `3407`，完整处理轻量调参交付的 `5` 个候选。
- 全量调参冻结轻量阶段确定的数据参数、标签、窗口和模型结构，只调整批次大小、学习率、优化器参数和调度器参数，并对 `5` 个候选逐一完成回测。
- 所有调参首先评价训练可持续性，至少比较 `val_loss` 改进次数、是否跑满 `120` 个 epochs、最佳轮次与结束轮次的距离，再结合 `val_loss`、`test_loss` 和回测结果排序。

## 主要变化

1. 把轻量调参的训练上限从 `60` 统一为 `120`，早停忍耐从 `8` 统一为 `9`，并把约十分之一样本改成明确的十分之一完整实体样本。
2. 把全量调参的训练上限从 `600` 统一为 `120`，早停忍耐从 `20` 统一为 `9`，完整数据边界保持不变。
3. 把所有调参实验的共同随机种子固定为配置值 `3407`。
4. 把候选数量从 `3` 统一为 `5`，取消全量调参只先训练第 `1` 名并等待确认的分段门禁。
5. 轻量调参覆盖数据参数、模型结构参数和训练参数，并为全部 `5` 个候选执行消融实验和回测。
6. 全量调参只搜索批次大小、学习率、优化器参数和调度器参数，保持候选的数据与模型身份不变，并逐一回测全部 `5` 个候选。
7. 候选排序首先检查训练可持续性，再比较损失和业务回测结果。
