## 当前实现要求

- 先完成中文源和中文包，再同步英文源和英文包。
- `$cx-doc` 的双语技能资产提供结构等价的默认项目 `AGENTS.md` 模板，并随技能镜像安装。
- 中英文示例项目都提供按真实目标、语言工具链和通用包填写的根目录 `AGENTS.md`，并包含可执行的通用包教程。
- 包清单包含 `$cx-doc` 和唯一交付审查入口 `$cx-review`。
- 单一来源校验器识别 `docs/cx/docs/`、`docs/cx/notes/` 和当前临时变更结构。
- `$cx-tdd` 承担唯一测试先行主流程；`$cx-pytorch-tdd` 和 `$cx-rust-tdd` 只补充各自语言的工具、布局、真实数据夹具和特有检查。
- 测试禁止表达式与表达式比较，也禁止完整有序集合的列表式比较；默认分别检查首项、末项和一个中间项，右侧使用已确认常量。
- Python 数据测试由 `tests/__init__.py` 一次读取测试数据库并共享真实对象；Rust 数据测试由统一夹具模块使用一次初始化机制共享真实对象。
- 除非用户当前请求明确要求，Python 和 Rust 测试都禁止使用模拟对象或伪造数据访问层。
- `$cx-pytorch-tdd`、`$cx-pytorch-quick-hpo` 和 `$cx-pytorch-full-hpo` 把系统内存与显存评估作为任何实验训练的第一道门禁，启动前记录预计峰值和安全余量。
- 预计资源占用较多或耗时较长的训练在 Codex App 之外的外部终端启动，持续写入文件型日志；人工智能代理通过读取日志文件分析，不依赖终端特殊处理。
- `$cx-pytorch-quick-hpo` 表达轻量调参：固定使用十分之一完整实体样本、`120` 个训练 epochs 上限、早停忍耐 `9` 和配置种子 `3407`，承担数据参数、模型结构参数和训练参数的主要选择，最终保留 `5` 个候选并分别完成消融实验和回测。
- `$cx-pytorch-full-hpo` 固定使用全部样本、`120` 个训练 epochs 上限、早停忍耐 `9` 和配置种子 `3407`；保持轻量阶段的数据参数、标签、窗口和模型结构不变，只调整批次大小、学习率、优化器参数和调度器参数，并逐一训练、测试和回测 `5` 个候选。
- 两个调参技能都先评价训练可持续性，记录 `val_loss` 改进次数和比例、是否跑满 `120` 个训练 epochs、最佳轮次及其距结束轮次的轮数和比例；可持续性相近时再比较 `val_loss`、`test_loss` 和回测结果。
- 所有编码规则绝对禁止顺手处理当前任务之外的问题；任务范围由用户当前请求、当前用例路径、原任务和未完成变更共同界定。
- 任务外问题只能列入包含文件、位置、现象、风险和建议的待授权清单；只有用户对具体清单项明确授权后才能修改，代理推测和审查意见不构成授权。
- 安装脚本继续从远端 `main` 更新全局技能和 `AGENTS.md`。

## 验证方式

- 运行中文包和英文包验证工具。
- 比较公共技能源与安装包镜像。
- 比较 `$cx-doc` 双语项目规则模板资产与安装包镜像。
- 快速校验中英文 `$cx-tdd`、`$cx-pytorch-tdd`、`$cx-rust-tdd` 和 `$cx-workflow` 技能目录。
- 搜索主流程重复、旧 Rust 路由、`pytest`、重复数据库读取和默认模拟测试等冲突说法。
- 搜索三个 PyTorch 技能的资源安全门禁、外部终端和文件型日志规则，确认中英文语义等价。
- 搜索轻量与全量调参技能的样本边界、`120` 个训练 epochs 上限、早停忍耐 `9`、配置种子 `3407`、训练可持续性、`5` 个候选、消融实验、允许调整的全量参数和逐候选回测，确认中英文语义等价且当前约定唯一。
- 搜索全局规则、`$cx-workflow` 和 `$cx-review` 的任务边界、绝对禁止、待授权清单和明确授权规则，确认中英文语义等价。
- 推送 `main` 后运行中文安装脚本并核对安装结果。
