# 设计：维护 cx 工作流

## 文件范围

- 中文技能源：`SKILLS/zh/`
- 英文技能源：`SKILLS/en/`
- 安装包镜像：`packages/zh/`、`packages/en/`
- 项目规则：根目录 `AGENTS.md`、`$cx-doc` 的双语 `assets/AGENTS.md`
- 项目级说明：`README.zh-CN.md`、`README.md`

## 功能入口

- `$cx-workflow`：先读取项目 `AGENTS.md` 和当前领域教程并搜索已登记通用能力，再分流当前工作。
- `$cx-story`：维护唯一当前用例、设计和固定任务集合。
- `$cx-changelog`：登记、提交和删除临时变更文件。
- `$cx-doc`：维护通用包调用者教程、专题文档、研究笔记和项目 `AGENTS.md` 教程导航。
- `$cx-tdd`：统一执行所有语言的测试先行主流程和断言纪律。
- `$cx-pytorch-tdd`：只补充 Python、PyTorch 和 Lightning 的测试工具、布局、共享真实数据和张量检查。
- `$cx-pytorch-quick-hpo`：轻量调参固定使用十分之一完整实体样本、明确训练 `120` 个 epochs、默认早停忍耐 `20` 和配置种子 `3407`，承担数据参数、模型结构参数和训练参数选择；候选形成阶段只搜索 `3–20` 天窗口，确定 `5` 个候选后再对当前最佳候选执行 `30`、`40`、`50`、`60` 天长窗口扩展实验，最后分别执行消融实验和回测。
- `$cx-pytorch-full-hpo`：全量调参固定使用全部样本、明确训练 `120` 个 epochs、默认早停忍耐 `20` 和配置种子 `3407`，冻结轻量阶段最终确定的数据与模型参数，只调整批次大小、学习率、优化器参数和调度器参数，并逐一训练、测试和回测 `5` 个候选。
- `$cx-rust-tdd`：只补充 Rust 内置测试、共享真实数据夹具和 `cargo` 检查。
- `$cx-review`：执行交付物质量审查和完成证据门禁。

## 可复用能力

- 复用现有双语技能目录、包清单和安装脚本。
- 复用 `$cx-doc` 技能资产分发默认项目 `AGENTS.md` 模板，不建立第二份安装模板。
- 复用现有 `changes/` 文件结构承载当前工作指令。
- 复用 `cx-review` 审查代理执行统一交付审查。

## 设计决策

| 决策 | 理由 | 影响 |
| --- | --- | --- |
| 正式文档只写当前状态 | 避免人工智能代理读取失效方案 | 未完成变更文件只服务当前工作 |
| 已有 story 的任务文件身份固定 | 避免同一工作产生大量增删改任务 | 实现方式变化时原地修改任务正文 |
| 完成后删除变更文件 | Git 已保存变更登记和删除历史 | `changes/` 只显示当前未完成工作 |
| 一个技能承担完整审查 | 保持单一审查入口 | `$cx-review` 使用两个连续审查阶段 |
| 通用包教程集中在 `docs/cx/docs/` | 让调用者从真实目标、入口和步骤理解现有能力 | 每个通用包拥有独立编号教程，其它文档只链接 |
| 项目 `AGENTS.md` 导航领域教程 | 让全局规则与项目事实分离，并在实现前完成知识门禁 | 模板随 `$cx-doc` 分发，项目按目标、语言、工具链和通用包填写 |
| 研究结论集中在 `docs/cx/notes/` | 让研究回答具体问题并可复用 | 研究笔记使用流畅中文和必要来源 |
| `$cx-tdd` 统一测试先行流程 | 避免语言技能重复并分叉测试步骤和断言规则 | 语言技能只保留工具链、布局、真实数据夹具和语言特有检查 |
| 测试期望值只使用常量事实 | 避免测试重复实现生产表达式或整表比较 | 有序集合只检查首项、末项和一个中间项，除非用户明确要求全量验证 |
| 数据测试集中加载真实测试数据库 | 避免每个测试重复读取或用模拟数据替代业务事实 | Python 在 `tests/__init__.py` 初始化，Rust 在统一夹具模块中一次初始化 |
| 实验训练先执行资源安全门禁 | 避免系统内存、显存或终端输出负载造成系统崩溃，尤其避免 Codex App 崩溃 | 启动前记录资源估算和安全余量，高资源或长耗时任务使用外部终端与文件型日志 |
| 轻量阶段承担主要选择，全量阶段只收敛训练参数 | 避免完整数据反复搜索数据参数和模型结构，控制全量训练成本并保持候选身份可比较 | 轻量阶段产出带消融和回测的 `5` 个候选；全量阶段只调整批次大小、学习率、优化器和调度器参数 |
| 调参以训练可持续性作为候选准入条件 | 单次最低损失或单次高回测不能证明训练仍在稳定改善 | 正常候选必须跑满 `120` 个 epochs，并在第 `101` 至 `120` 轮仍产生 `val_loss` 新最佳值；全部方案都不满足时先按完成轮数从多到少回退，再比较末段改进、损失和回测 |
| 滑动窗口分成基础搜索与长窗口扩展 | 长窗口会显著改变显存、训练耗时和 batch 预算，过早混入会放大候选形成成本 | 候选形成阶段限定 `3–20` 天，确定 `5` 个候选后只对当前最佳候选固定其它参数执行 `30`、`40`、`50`、`60` 天四次扩展实验 |
| 循环学习率与早停共同配置 | 升温和降温尚未完成就早停会错误淘汰可持续方案 | 默认早停忍耐 `20`；使用升温和降温循环时显式保存循环参数，patience 覆盖一个完整循环并尽可能不超过 `30` |
| 绝对禁止顺手处理任务外问题 | 防止代理把推测、偏好或审查意见误当成用户需求 | 任务外问题只进入待授权清单，用户对具体清单项明确授权后才能修改 |

## 非目标

- 不保存完成变更档案。
- 不为同一任务建立“新增”“修改”“删除”任务链。
- 不设置第二个交付审查入口。
- 不把专题知识重复写入任务、变更和包内说明。
- 不在项目 `AGENTS.md` 复制全局规则或保留未填写模板项。
- 不把代理发现的任务外问题自动纳入实现、清理或审查修复范围。

## 验证方式

- 校验中文和英文技能集合一致。
- 校验 `$cx-doc` 的双语项目规则模板资产与安装包镜像一致。
- 校验项目 `AGENTS.md` 覆盖项目目标、语言工具链、全部通用包教程链接和领域先读门禁。
- 校验安装包清单与技能目录一致。
- 校验仓库只有 `$cx-review` 一个交付审查技能入口。
- 校验 `$cx-tdd` 是唯一测试先行主流程，两个语言技能没有复制主流程。
- 校验测试断言、真实数据库共享和默认禁止模拟测试的中英文规则等价。
- 校验三个 PyTorch 技能都先执行内存与显存评估，并统一使用外部终端和文件型日志承载高资源或长耗时训练。
- 校验轻量与全量调参技能的中英文约定都包含明确 `120` 个训练 epochs、默认早停忍耐 `20`、循环调度显式参数及 patience 尽可能不超过 `30`、配置种子 `3407`、正常候选准入条件、全部方案不满足时的完成轮数回退和 `5` 个候选；轻量调参另含十分之一完整实体样本、`3–20` 天基础窗口、候选确定后的 `30`、`40`、`50`、`60` 天扩展实验、逐候选消融和逐候选回测，全量调参另含全部样本、限定训练参数搜索空间和逐候选回测。
- 校验全局规则、`$cx-workflow` 和 `$cx-review` 绝对禁止顺手处理任务外问题，并要求待授权清单和用户具体授权。
- 搜索正式文档中是否要求保存完成变更或过程差异。
- 运行项目现有包验证工具；本次未声明单元测试。
