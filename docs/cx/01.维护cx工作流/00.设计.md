# 设计：维护 cx 工作流

## 文件范围

- 中文技能源：`SKILLS/zh/`
- 英文技能源：`SKILLS/en/`
- 安装包镜像：`packages/zh/`、`packages/en/`
- 项目规则：根目录 `AGENTS.md`、`$cx-doc` 的双语 `assets/AGENTS.md`
- 项目级说明：`README.zh-CN.md`、`README.md`

## 功能入口

- `$cx-workflow`：先读取项目 `AGENTS.md` 和当前领域教程并搜索已登记通用能力，再分流当前工作。
- `$cx-story`：维护唯一当前用例、设计和固定任务集合。
- `$cx-changelog`：登记、提交和删除临时变更文件。
- `$cx-doc`：维护通用包调用者教程、专题文档、研究笔记和项目 `AGENTS.md` 教程导航。
- `$cx-tdd`：统一执行所有语言的测试先行主流程和断言纪律。
- `$cx-pytorch-tdd`：只补充 Python、PyTorch 和 Lightning 的测试工具、布局、共享真实数据和张量检查。
- `$cx-rust-tdd`：只补充 Rust 内置测试、共享真实数据夹具和 `cargo` 检查。
- `$cx-review`：执行交付物质量审查和完成证据门禁。

## 可复用能力

- 复用现有双语技能目录、包清单和安装脚本。
- 复用 `$cx-doc` 技能资产分发默认项目 `AGENTS.md` 模板，不建立第二份安装模板。
- 复用现有 `changes/` 文件结构承载当前工作指令。
- 复用 `cx-review` 审查代理执行统一交付审查。

## 设计决策

| 决策 | 理由 | 影响 |
| --- | --- | --- |
| 正式文档只写当前状态 | 避免人工智能代理读取失效方案 | 未完成变更文件只服务当前工作 |
| 已有 story 的任务文件身份固定 | 避免同一工作产生大量增删改任务 | 实现方式变化时原地修改任务正文 |
| 完成后删除变更文件 | Git 已保存变更登记和删除历史 | `changes/` 只显示当前未完成工作 |
| 一个技能承担完整审查 | 保持单一审查入口 | `$cx-review` 使用两个连续审查阶段 |
| 通用包教程集中在 `docs/cx/docs/` | 让调用者从真实目标、入口和步骤理解现有能力 | 每个通用包拥有独立编号教程，其它文档只链接 |
| 项目 `AGENTS.md` 导航领域教程 | 让全局规则与项目事实分离，并在实现前完成知识门禁 | 模板随 `$cx-doc` 分发，项目按目标、语言、工具链和通用包填写 |
| 研究结论集中在 `docs/cx/notes/` | 让研究回答具体问题并可复用 | 研究笔记使用流畅中文和必要来源 |
| `$cx-tdd` 统一测试先行流程 | 避免语言技能重复并分叉测试步骤和断言规则 | 语言技能只保留工具链、布局、真实数据夹具和语言特有检查 |
| 测试期望值只使用常量事实 | 避免测试重复实现生产表达式或整表比较 | 有序集合只检查首项、末项和一个中间项，除非用户明确要求全量验证 |
| 数据测试集中加载真实测试数据库 | 避免每个测试重复读取或用模拟数据替代业务事实 | Python 在 `tests/__init__.py` 初始化，Rust 在统一夹具模块中一次初始化 |

## 非目标

- 不保存完成变更档案。
- 不为同一任务建立“新增”“修改”“删除”任务链。
- 不设置第二个交付审查入口。
- 不把专题知识重复写入任务、变更和包内说明。
- 不在项目 `AGENTS.md` 复制全局规则或保留未填写模板项。

## 验证方式

- 校验中文和英文技能集合一致。
- 校验 `$cx-doc` 的双语项目规则模板资产与安装包镜像一致。
- 校验项目 `AGENTS.md` 覆盖项目目标、语言工具链、全部通用包教程链接和领域先读门禁。
- 校验安装包清单与技能目录一致。
- 校验仓库只有 `$cx-review` 一个交付审查技能入口。
- 校验 `$cx-tdd` 是唯一测试先行主流程，两个语言技能没有复制主流程。
- 校验测试断言、真实数据库共享和默认禁止模拟测试的中英文规则等价。
- 搜索正式文档中是否要求保存完成变更或过程差异。
- 运行项目现有包验证工具；本次未声明单元测试。
