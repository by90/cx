# cx 项目说明

## 项目目标

1. 以 `docs/cx` 中唯一的当前状态文档约束人工智能代理，让实现、设计、任务和专题知识始终反映项目现在有效的事实。
2. 以临时变更文件指导已有 story 的修改，让工作开始和完成都进入 Git，同时不在工作区长期保存开发过程历史。
3. 优先复用项目已经登记的通用能力，让人工智能代理通过项目 `AGENTS.md` 先读取当前领域的调用者教程，再搜索真实代码入口。
4. 以统一交付审查同时检查交付物质量和完成证据，让任务只在实现、文档、验证和状态全部一致时完成。

## 关键术语

- 当前状态文档：只描述项目现在有效的目标、设计、任务、接口和结论，不保存被替代方案或新旧差异。
- 临时变更文件：位于主成功场景 `changes/` 下、只在工作未完成期间存在的工作指令；完成后删除，历史由 Git 保存。
- 调用者教程：位于 `docs/cx/docs/` 下，按序号命名，从使用者角度说明通用包的公共入口、最小示例、操作步骤、预期结果和失败处理。
- 项目 `AGENTS.md`：位于项目根目录，根据项目目标、语言、工具链和通用包补充全局规则，并导航各领域必须先读的教程。
- 专题文档：位于 `docs/cx/docs/` 下，按序号命名，说明可复用技术过程的稳定知识。
- 研究笔记：位于 `docs/cx/notes/` 下，按研究问题命名，保存针对具体问题的研究结论和通俗说明。

## 设计约束

- 正式文档只保留当前状态。
- 已有 story 的任务文件数量和文件身份保持不变。
- 代码缺陷、设计改向和任务实现方式变化都先登记临时变更文件。
- 每个通用包都有独立调用者教程，并由项目 `AGENTS.md` 登记领域、教程链接、公共入口和先读条件。
- 通用能力搜索先读取项目 `AGENTS.md` 中当前领域的教程，再搜索源码和实际调用点。
- Git 提交、标签和发布记录承担历史保存职责。
- 除非用户在当前请求中明确要求具体校验或异常处理，程序异常保持原始类型、信息和调用栈并自然中止程序，不通过代理自行增加的校验、捕获、包装或兜底改变行为。
- 开发实现只保留当前最新意图和当前入口，不保存旧接口、兼容分支、适配层、旧行为或相关痕迹。
