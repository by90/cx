# cx Python 与 PyTorch 测试补充

## 资源安全前置门禁

任何包含实验性训练的测试工作，第一步必须评估预计系统内存占用和显存占用，再决定是否启动。评估至少记录当前可用系统内存、当前可用显存、预计峰值、保留给操作系统和 Codex App 的安全余量，以及允许启动或必须缩小实验的结论；可以使用模型结构、张量形状、批量大小和最小试运行取得估算，但不得先启动完整训练再观察是否耗尽资源。

本技能内任何预计资源占用较多或耗时较长的任务都必须在 Codex App 之外的外部终端启动，使任务进程和大量输出不由 Codex App 内置终端承载。任务进度、标准输出、标准错误、系统内存、进程内存、显存占用和错误必须持续写入文件型日志；人工智能代理只通过读取这些日志文件分析任务状态，不依赖终端滚动输出、终端控制序列或其它特殊终端处理。此门禁用于避免系统资源耗尽和系统崩溃，尤其避免 Codex App 因任务进程或终端输出负载而崩溃。

## 边界

本技能不定义测试先行流程。先完整执行 `$cx-tdd`，再把本技能的 Python、PyTorch 和 Lightning 规则叠加到同一个测试循环中。

没有明确测试要求时，不使用本技能，不创建、不修改、不运行单元测试。

## Python 工具与布局

- 使用项目级 `uv` 环境运行 Python、格式检查和测试。
- 使用 Python 内置 `unittest`，禁止引入 `pytest`；只有用户当前请求明确要求时才例外。
- 源文件 `src/<subsystem>/xx.py` 只对应测试文件 `tests/<subsystem>/xx_test.py`，不得用一个大测试文件覆盖多个源文件，也不得为同一源文件拆出多个测试文件。
- `src/` 及其子目录保留空白 `__init__.py`。`tests/` 根目录的 `__init__.py` 是共享测试数据库初始化入口，不得留空；`tests/` 的其它包目录保留空白 `__init__.py`。
- 源码和测试使用仓库根包绝对导入，不得修改 `sys.path`。
- 使用 `uv run python -m unittest discover -v -s ./tests -p "*_test.py" -t .` 验证测试发现；开发循环先运行更窄的测试方法或测试类。
- 修改 Python 源码或测试后，使用项目 `uv` Python 运行 Black 默认格式检查。
- 只有创建或重建环境时才核对 Python、PyTorch 和 CUDA 官方稳定组合；接口行为可能随版本变化时只查官方文档。

## 真实测试数据

- 数据相关测试必须使用测试数据库中的真实数据，禁止在测试文件中临时制造替代数据。
- `tests/__init__.py` 负责一次打开测试数据库、读取固定测试范围并构造共享领域对象；各 `xx_test.py` 只导入并复用这些共享对象。
- 禁止每个测试文件、测试类或测试方法重复打开数据库、重复查询同一测试范围或重复构造同一领域对象。
- 测试数据库或所需真实记录缺失时，报告前置条件缺失，不用内存数据、伪数据库或模拟访问层替代。
- 除非用户当前请求明确要求，禁止 mock 测试。

## PyTorch 与 Lightning 检查

- 测试保持确定、极小并优先使用 CPU；只有目标行为本身是 CUDA 或其它设备行为时才使用对应设备。
- 张量行为按当前要求断言形状、数值类型和设备；数值类型与配置子系统中的类型对象比较，只有配置默认值测试可以直接断言框架类型常量。
- 依照 `$cx-tdd` 分别比较第一个、最后一个和任意一个中间张量元素与已知常量，不比较完整张量。
- 只在当前行为需要时覆盖空输入、单项、多项、变长、填充、掩码、长度、确定性或梯度。
- Lightning 编排测试使用极小批次、小模型和有限批次；禁止把长训练循环放进单元测试。

## 输出

除 `$cx-tdd` 要求的记录外，补充记录 `unittest` 命令、`uv` 环境、共享测试数据库入口，以及张量的形状、数值类型和设备验证结果。
