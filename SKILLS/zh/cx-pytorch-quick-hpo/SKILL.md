# cx PyTorch 快速调参

## 目的

用较小但完整的样本快速找到值得进入全量调参的配置候选。快速调参优先解决数据字段组合、窗口长度和标签定义，再处理训练超参、optimizer、scheduler，最后才搜索模型结构容量和复杂度。

## 铁律

1. 训练脚本必须能单独运行，并在默认配置下完成一次训练；调参脚本只能通过 typed config hook 在内存中复制并修改配置，再把该配置对象传给训练入口。
2. 回测脚本必须能单独运行，并在默认配置下完成一次回测；调参脚本只能通过内存配置对象调用回测入口，不能直接修改当前回测配置文件。
3. 调参脚本保存结果时，只能把 trial 配置、best recipe、指标和回测配置持久化到输出目录；禁止直接改写项目当前配置、默认配置或用户手动维护的配置文件。
4. 快速调参使用约十分之一样本，但抽样单位必须是完整实体。比如 2000 只股票的日线数据，应随机选择约 200 只股票，并保留这些股票的完整日线。
5. 快速调参前，必须针对所有候选模型研究字段组合中每个字段相对标签的正贡献、负贡献和噪声嫌疑，判断重要字段、次重要字段、重复字段和很可能多余字段。
6. 字段研究必须写入目标文档集的数据处理部分；如果项目已有 `series` 数据处理目录，优先写入 `docs/<feature_group>/series/` 下的说明文档。
7. 字段研究必须明确日线基础字段、市场强度指标、位置指标和排名指标的语义边界：日线字段是基础，市场强度指标衡量横向整体强度，位置指标衡量窗口在时间轴上的位置，排名指标衡量横向特征比较。
8. 对排名字段必须专门研究：它们通常有对应基础字段，必须判断基础字段和排名字段同时使用是否不妥、单独使用排名字段是否足够、排名字段与位置字段组合是否可行。
9. 快速调参时，必须先给出启用哪些字段或特征的明确建议，并用一个固定模型验证不同字段组合的表现，再把字段组合建议写入字段研究文档。
10. 搜索顺序必须是：字段组合、窗口长度和标签定义；训练超参、optimizer 和 scheduler；模型结构参数、容量和复杂度。
11. 快速训练计划上限是 60 个 epochs，early stopping 监控 `val_loss`，patience 一般设为 8；触发早停时必须停止，但自然跑满 60 个 epochs 且仍有稳定改善是更好的候选信号。
12. 必须记录 `val_loss` 改进次数相对 epochs 数量的比例；比例过低说明收敛质量可疑，不能只看最终 `val_loss`。
13. 分类任务默认把 `val_loss`/cross-entropy 作为训练 objective、调参和早停监控指标；accuracy、precision、recall、F1、AUC 等只作为业务评估指标同步记录。
14. 多模型项目中，每个模型都可能有自己唯一的最优字段组合、窗口长度、标签和容量配置；快速调参必须保留每个模型的模型内最佳候选，不得过早只保留一个跨模型全局排名。

## 必须执行的流程

1. 先确认目标文档集已经定义业务目标、数据边界、标签目标和验证方式；缺少时回到 `$cx-workflow` 推荐 `$cx-bdd`、`$cx-research` 或 `$cx-timeseries-modeling`。
2. 先用 `$cx-common-module` 检查训练入口、回测入口、config hook、recipe 和输出持久化是否已有公共入口；没有时先定义调用模型。
3. 用 `$cx-pytorch-tdd` 验证训练脚本、回测脚本、config clone/override hook、trial 到 config 映射和输出持久化路径。
4. 先做字段贡献研究和小实验，再确定快速调参字段候选；不要在字段语义未清楚时直接搜索模型容量。
5. 用一个固定模型验证字段组合、窗口长度和标签定义，得到数据 recipe 候选。
6. 在数据 recipe 候选稳定后，再搜索 batch size、learning rate、weight decay、optimizer 和 scheduler。
7. 最后搜索模型结构参数，例如 hidden size、layers、dropout、encoder length、模型容量和复杂度。
8. 搜索后必须复跑 best recipe，并把 best trial、top candidates、每个模型的模型内 best recipe、失败 trial、pruned trial、随机种子、数据版本、字段建议和收敛证据写入目标文档集。

## 验证证据

- 训练脚本和回测脚本可以在默认配置下独立运行。
- 调参脚本只在内存中修改配置，输出目录包含被保存的 trial config 和 best recipe，当前配置文件未被改写。
- 十分之一完整实体样本的抽样单位、实体数量、覆盖范围和随机种子。
- 字段贡献、负贡献、噪声嫌疑、重复字段和多余字段的研究结论。
- 字段组合建议，以及固定模型验证不同字段组合的结果。
- 窗口长度、标签定义、训练超参、optimizer、scheduler 和模型结构搜索空间。
- 60 epochs / patience=8 的执行情况、早停 epoch、best `val_loss`、`val_loss` 改进次数和改进比例。
- 每个模型的模型内最佳候选、推荐进入全量调参的候选配置，以及排除其他候选的理由。
